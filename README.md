# KittyHP

Base del sistema para registrar reportes de reparación con NestJS, Angular standalone y MySQL.

## Estructura objetivo

- `backend/`: API NestJS con módulos por dominio.
- `frontend/`: app Angular standalone.
- `database/`: scripts SQL y estructura inicial.

## Tabla base

La tabla inicial modela la hoja que compartiste. Se conserva un campo JSON para no perder columnas variables o importaciones futuras.
